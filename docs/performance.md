# Performance

Benchmarks compare the native Delphi implementation with the pinned LZMA SDK
26.01 and xz-utils tools used by the test suite.

The public README contains the current headline throughput and ratio table.
Regenerate detailed benchmark data with:

```powershell
.\tests\install-qa-tools.ps1
.\tests\run-performance.ps1 -WorkRoot <SSD_WORKROOT>
```

The benchmark work root must be an owned scratch directory outside the
repository. Use a local SSD path for normal runs. A RAM-backed path can be used
only when explicitly selected by `-RamWorkRoot`.

## Outputs

`tests/run-performance.ps1` writes data-only files under ignored
`artifacts/perf/`:

- `lzma2-benchmark.csv`
- `lzma2-benchmark.json`
- `lzma2-benchmark.samples.csv`
- `lzma2-benchmark.summary.json`

The benchmark artifacts record:

- input and output sizes;
- compression ratio;
- elapsed time;
- throughput in MiB/s and MB/s;
- thread count;
- dictionary size;
- check type;
- peak process memory when available;
- tool versions and normalized command options.

Graph images are not generated by release validation. If charts are needed,
generate them separately from the data files.

## Notes

Compression output is format-compatible, but not expected to be byte-identical
to the SDK output. Compare compressed size, ratio, throughput, and successful
round trips instead.

Short smoke runs can be noisy. Use release-size corpora and repeated samples
when making performance claims.
